# クライアント配布（案A: 共有フォルダ＋自動更新）

ClickOnce はフルの Visual Studio が必要なため、VS不要で同等（リンク配布＋自動更新）を実現する方式。

## 配布共有のレイアウト
```
\\192.168.1.242\MTlock関連\system\SecureFileExplorer\
├─ app\                      ← 発行したクライアント一式（self-contained）
├─ version.txt               ← 現在の配布バージョン（例: 1.0.0）
├─ Launch.ps1                ← ランチャー本体（自動更新ロジック）
├─ 技術部ファイルビューアー.cmd   ← 起動用（ショートカットの実体）
└─ 技術部ファイルビューアー.lnk   ← 配布するショートカット（ユーザーはこれを使う）
```

## 仕組み（自動更新）
ユーザーがショートカットを実行 → `Launch.ps1` が動作:
1. 共有の `version.txt` を読む
2. ローカル `%LOCALAPPDATA%\SecureFileExplorer\<version>\` に未取得なら、共有 `app\` からコピー
3. ローカルのexeを起動（バージョンごとに別フォルダ＝実行中exeの上書き問題なし）

→ アプリを更新したいときは、`app\` を新版で差し替え、`version.txt` を上げるだけ。
   全員が次回起動時に自動で最新版になる。

## 配り方
- ユーザーには `技術部ファイルビューアー.lnk`（ショートカット）を配る（メール/共有/デスクトップ配置）。
- 初回起動時のみローカルへ約160MBコピー（LAN内なら数秒）。以降は即起動。

## 運用更新の手順（開発者）
1. `dotnet publish` でクライアントを再発行
2. 共有 `app\` を差し替え
3. `version.txt` を新バージョンに更新
